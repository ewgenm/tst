# 📘 ТЗ №1: BACKEND (Laravel 13)
**Проект:** TaskSync Team  
**Стек:** Laravel 13, PHP 8.3+, PostgreSQL, Redis, Sanctum, Reverb, OpenAI SDK  
**Локальный домен:** `tst.test` (Laravel Herd)  
**Источник истины:** ТЗ №0 v1.2

## 1. Цель и Архитектура
Создать API-First бэкенд, строго следующий контракту ТЗ №0. Архитектура: **Controllers → FormRequests → Services/Actions → Models → API Resources**. Минимум бизнес-логики в контроллерах.

## 2. Настройка проекта и окружение
```bash
# Инициализация
laravel new tasksync-backend --branch=13.x
cd tasksync-backend
composer require laravel/sanctum laravel/reverb spatie/laravel-query-builder spatie/eloquent-sortable openai-php/client
```

**`.env` (ключевые переменные):**
```env
APP_NAME="TaskSync Team"
APP_URL=http://tst.test
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_DATABASE=tasksync_dev
REDIS_HOST=127.0.0.1
BROADCAST_CONNECTION=reverb
REVERB_APP_ID=tasksync
REVERB_APP_KEY=tasksync-key
REVERB_APP_SECRET=tasksync-secret
```

## 3. База данных и Миграции
- Строго следовать схемам из ТЗ №0.
- Использовать `uuid` или `bigIncrements` для ID (рекомендуется `bigIncrements` для простоты сортировки).
- Обязательно добавить `softDeletes()` для `tasks` и `projects`.
- Индексы: `tasks(project_id, status, position)`, `tasks(assignee_id, status)`, `tasks(due_at)`.
- Связи: `cascadeOnDelete()` для зависимых записей, `nullOnDelete()` только для `parent_task_id` (но по ТЗ меняем на `cascadeOnDelete()` для подзадач).

## 4. Аутентификация и ACL
- **Sanctum:** Stateful SPA auth. Включить `EnsureFrontendRequestsAreStateful::class` в `api` middleware.
- **CORS:** Разрешить `http://tst.test`, `http://localhost:3000`.
- **Policies:** Создать `ProjectPolicy`, `TaskPolicy`, `CommentPolicy`. Реализовать матрицу из ТЗ №0.
- **Gates:** Для глобальных проверок (например, `can-import`).

## 5. Реализация API (Пошагово для ИИ)

### 5.1. Структура ответа
Использовать `API Resource` коллекции. Формат:
```json
{
  "success": true,
  "data": { ... },
  "meta": { "timestamp": "..." }
}
```
Создать базовый `BaseResource` и `BaseCollection`, которые автоматически оборачивают ответ.

### 5.2. Модуль: Tasks
- `TaskController`: index (с фильтрацией через `spatie/laravel-query-builder`), store, show, update, destroy, restore, complete.
- `StoreTaskRequest`, `UpdateTaskRequest`: валидация по ТЗ №0.
- `CompleteTaskAction`: 
  1. Помечает задачу `done`.
  2. Если `is_recurring`, создает копию с новым `due_at` (парсинг RRULE).
  3. Отправляет событие `TaskUpdated` через Reverb.
- `TaskResource`: маппинг полей, подгрузка `project`, `assignee`, `tags`, `attachments_count`.

### 5.3. Модуль: Projects
- `ProjectController`: CRUD, archive, invite, leave, members.
- `InviteMemberAction`: создает запись в `project_members` со `status='pending'`. Отправляет email/уведомление.
- `AcceptInviteAction`: меняет `status` на `active`.
- Валидация: владелец проекта всегда `auth()->id()` при создании. Роль `owner` в `project_members` запрещена.

### 5.4. Модуль: Attachments & Import/Export
- `AttachmentController`: upload (multipart), list, delete. Использовать `Storage::disk('local')` или `s3`.
- `ExportService`: генерация CSV/JSON через `Maatwebsite/Laravel-Excel` или нативный `fputcsv`.
- `ImportService`: парсинг JSON/CSV, создание задач/проектов с транзакциями.

### 5.5. Модуль: AI & Integrations
- `AiService`: обертка над OpenAI API.
  - `generateSubtasks(string $taskDesc): array`
  - `suggestPlan(array $tasks): array`
- Rate limiting: 20 req/hour через `RateLimiter::for('ai', ...)`.
- Интеграции: Webhook endpoint `/webhooks/telegram`, OAuth flow для Google Calendar.

### 5.6. Real-time (Reverb)
- События: `TaskCreated`, `TaskUpdated`, `TaskDeleted`, `CommentAdded`, `NotificationSent`.
- Каналы: `private-user.{id}`, `private-project.{id}`.
- Broadcast на `ShouldBroadcastNow` для мгновенной отправки.

## 6. Тестирование
- **Feature Tests:** `TaskApiTest`, `ProjectApiTest`, `AuthTest`.
- Проверка статусов кодов, структуры JSON, прав доступа (ACL).
- Mock OpenAI ответа в тестах.

## 7. 🤖 Инструкции для ИИ-агента (Backend)
1. **Генерируй по одному модулю за раз.** Начни с `Auth → Projects → Tasks → Attachments → AI`.
2. **Строго соблюдай ТЗ №0.** Не меняй имена полей, типы данных или структуру ответа.
3. **Используй типизацию PHP 8.3:** `readonly class`, `enum`, типизированные свойства.
4. **Валидация только в FormRequest.** Контроллеры должны быть тонкими.
5. **Каждый эндпоинт покрывай Feature-тестом** перед генерацией следующего.
6. **Prompt-шаблон для ИИ:**
   ```
   Создай модуль [Название] согласно ТЗ №0 v1.2.
   Включи: Migration, Model, Controller, FormRequests, API Resource, Policy, Service/Action.
   Следуй архитектуре Laravel 13. Добавь Feature-тесты.
   Не меняй структуру JSON ответа.
   ```
